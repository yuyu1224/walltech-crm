## 🛠 1. 如何运行本项目 (Quick Start)

本项目采用纯前端架构，无需配置复杂的后台数据库，开箱即用。

### 环境要求
- Node.js (推荐 v16+ 版本)
- 现代浏览器 (Chrome / Edge / Safari 等均可)

### 运行步骤
1. **安装依赖**：
   在项目根目录下，打开终端并运行：
   ```bash
   npm install
   ```
2. **启动本地开发服务器**：
   ```bash
   npm run dev
   ```
3. **访问应用**：
   根据终端输出的提示，在浏览器中打开链接（通常是 `http://localhost:5173/`）。

### 数据持久化说明
为了保证最流畅的演示体验，本 Demo 的数据存储（客户名单、跟进日志、KPI 设定参数）均依托于浏览器原生的 `LocalStorage`。
- **免配置**：您可以随意增删改测试，刷新页面数据也不会丢失。
- **一键重置**：如果想恢复初始预置的 Mock 数据，只需在浏览器中按 `F12` 打开开发者工具，进入 `Application` (或存储) 面板，清空 Local Storage 或无痕模式打开后刷新即可。

## 🏗 2. 技术架构与选型 (Tech Stack)

考虑到这是一个“高交互、重体验”的敏捷呈现 Demo，我选用了以下现代化前端技术栈：

- **核心框架**：`React 18` (构建清晰的组件化 UI)
- **构建工具**：`Vite` (提供极速的冷启动和模块热更新，拒绝漫长等待)
- **全局样式**：`Tailwind CSS` (使用原子化 CSS 快速还原出现代化、高质感的 SaaS B端企业级界面)
- **图标矢量图**：`lucide-react` (提供轻量且一致的商务风格图标)

```text

├── tailwind.config.js      # Tailwind 配置文件

```

### 单文件组件说明 (`src/App.jsx`)
为了降低评委审阅代码寻找逻辑的成本，本 Demo 将核心逻辑收拢在 `App.jsx` 单文件中，自上而下包含以下三个子模块：

1. **`ToastContainer` (消息提示总线)**：
   - 托管全局的轻量化提示（如“创建成功”、“输入格式错误”），取代原生 `alert`，提供更好的交互质感。
2. **`KPIDashboard` (销售效能看板组件)**：
   - 即“主管/管理层视角”。实时拉取跟进数据，包含图表化的指标进度条；内置了参数干涉面板，允许主管实时下发“总跟进下限 / 拜访下限”等动态参数，计算排行榜。
3. **`CustomerList` (客户资源池组件)**：
   - 即“销售大厅”。实现了纯前端的模糊搜索、主营业务过滤、列表正反轴排序。并在表头汇聚今日待跟进预警数字。
4. **`CustomerDetail` (跟进日志时间轴组件)**：
   - 支持高频的“新增跟进记录”（具有日历穿梭拦截防呆）；右侧呈现瀑布流式交互轴，并包含智能化的超时未跟进红字告警。

## 🎮 4. 最佳功能寻宝动线 (Demo Guide)

为了节省您的时间，请按以下路径体验我在此 Demo 中埋下的“业务级校验”小心机：
1. **感受高频检索**：进入 [客户资源池]，直接通过右上角的下拉框筛选业务类型。
2. **触发防呆拦截**：点击 [详情>] 进入某个客户，在左侧填报表单的“下期计划提醒”中，故意选一个昨天的日期然后点击提交，体验顺滑的 Toast 拦截提示。
3. **查看视觉抓手**：在右侧时间轴滑动，寻找因为“下限提醒期过了”而自动变红报警的卡片。
4. **动大盘指标**：切到顶部 [效能看板] Tab，以主管身份点击 `⚙️主管视角`，将考核数字直接调高到 50，观察下方绿色进度条断崖式坠落——它实时反映了员工任务难度的增加。
