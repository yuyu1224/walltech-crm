## 1. 如何运行本项目

本项目采用纯前端架构，无需配置后台数据库，可直接运行。

### 环境要求
- Node.js（推荐 v16+ 版本）
- 现代浏览器（Chrome / Edge / Safari 等均可）

### 运行步骤
1. **安装依赖**：
   在项目根目录下，打开终端并运行：
   ```bash
   npm install
   ```
2. **启动本地开发服务器**：
   ```bash
   npm run dev
   ```
3. **访问应用**：
   根据终端输出的提示，在浏览器中打开链接（通常是 `http://localhost:5173/`）。

### 数据持久化说明
本 Demo 的数据存储（客户名单、跟进日志、KPI 设定参数）依托于浏览器原生的 `LocalStorage`：
- 可随意增删改测试，刷新页面数据不会丢失。
- 若想恢复初始预置的 Mock 数据，只需在浏览器中按 `F12` 打开开发者工具，进入 `Application`（或存储）面板，清空 Local Storage 或用无痕模式打开后刷新即可。

## 2. 技术架构与选型

考虑到这是一个交互性较强的演示 Demo，选用以下前端技术栈：

- **核心框架**：`React 18`（用于构建组件化 UI）
- **构建工具**：`Vite`（提供较快的冷启动和模块热更新）
- **全局样式**：`Tailwind CSS`（使用原子化 CSS 构建企业级界面）
- **图标矢量图**：`lucide-react`（提供商务风格图标）

```text

├── tailwind.config.js      # Tailwind 配置文件
└── vite.config.js          # Vite 构建配置
```

### 单文件组件说明 (`src/App.jsx`)
为方便查看代码逻辑，本 Demo 将核心逻辑收拢在 `App.jsx` 单文件中，自上而下包含以下子模块：

1. **`ToastContainer`（消息提示总线）**：
   托管全局提示（如“创建成功”“输入格式错误”），替代原生 `alert`。
2. **`KPIDashboard`（销售效能看板组件）**：
   即“主管/管理层视角”，实时展示跟进数据，包含指标进度条；内置参数调整面板，主管可下发“总跟进下限/拜访下限”等参数，计算排行榜。
3. **`CustomerList`（客户资源池组件）**：
   即“销售大厅”，实现纯前端的模糊搜索、主营业务过滤、列表排序，并在表头显示今日待跟进预警。
4. **`CustomerDetail`（跟进日志时间轴组件）**：
   支持新增跟进记录（含日期校验）；右侧呈现时间轴，并对超时未跟进的记录进行红色告警。

## 4. 功能体验路径

1. **左侧边栏导航与存量模块模拟**：
   - 切换左侧导航（如“市场营销”、“服务与支持”、“报表与分析”），查看提示“系统原有存量模块”，感受作为大型 CRM 系统的扩展性设计体验。
2. **业务检索与过滤（客户管理）**：
   - 点击左侧“客户管理”，进入“客户资源池”。体验顶部的快速模糊搜索以及业务类型（如：海运出口、国际空运）的下拉筛选。
3. **客户全景跟进（时间轴追踪）**：
   - 在客户列表中点击“详情”，进入客户全景视图。
   - 在左侧新增跟进记录。如果在“下次计划提醒”选择过去的日期提交，会触发逻辑报错拦截。
   - 提交成功后，在右侧的“历史交互轴”查看按时间排序的跟进节点。对于过去的超时提醒计划，系统会打上红色的告警标签。
4. **效能考核与排行榜分析（销售管理）**：
   - 点击左侧“销售管理”，进入“月度销售效能考核 (KPI)”看板（即主管视角）。
   - 点击右上角的“⚙️ 主管视角: 设定指标”，动态调高“月度总跟进下限”等参数，保存后可观察卡片进度条的实时自动计算和变化。
   - 查看底部的“本月销售绩效排行榜”，这里会自动汇总前面所有客户跟进表单中产生的“总跟进行动”数据，形成完整的数据闭环。
